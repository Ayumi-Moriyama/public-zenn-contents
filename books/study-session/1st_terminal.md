## CUIとGUI
### CUI
CUI とは「Character User Interface (キャラクター・ユーザー・インターフェース) 」の略称で、文字入力によってコンピューターを操作する方法
:::message
ターミナルはCUI
:::

私が思うメリット：
操作が早い。コマンドをメモ帳等に控えておけばコピペでいつも同じ操作ができる。
デメリット：
取り返しが効かない操作がある。打ち間違えた時に気づきにくい。

### GUI
GUIとは、「Graphical User Interface（グラフィカル・ユーザー・インターフェース）」の略であり、ユーザーの使いやすさを重視し、アイコンやボタンなどを用いて直感的にわかりやすくコンピューターに指令を出せるようにしたユーザーインターフェース
:::message
Soursetree（後述）はGUI
:::

私が思うメリット：
流れがわかりやすい。エラーメッセージを確認しやすい。
デメリット：
操作が手間。何度も同じ操作をする必要がある場合特に面倒。

## VS Codeでのターミナル操作
まずは、VS Codeのターミナルを開いてみましょう。
Macの場合　```command + j```

## 基本ルール
- 半角英字で入力
- 間は半角スペース

ターミナルのコマンド操作でできることはたくさんありますが、よく使うものを分類してみました。

1. 移動系
2. 内容確認系
2. 作成系
3. 削除系


### 移動系
- どこかへ移行したいときに入れるコマンド。
```
cd （移動先）
```
ターミナル操作では、　**現在いる場所がどこなのか？**　が重要になります。
今いる場所に対して操作を行う都合上、頻繁に移動する必要があるのでcdが一番よく使います。


### 内容確認系
- フォルダの中身を確認したりする際に入れるコマンド。
```
ls
```
- 隠しファイル含む全てのファイルを知りたい際には、 
```
ls -la
```

### 作成系
- ファイルを作成する
```
touch (作成するファイル名)
```

- フォルダ（ディレクトリといいます）を作成する
```
mkdir (作成するディレクトリ名)
```
**m**a**k**e **dir**ectory　の略

- コピーを作成
```
cp （コピーしたい既存のファイル名）（別のファイル名）
```
**c**o**p**y の略
::: message
コピーするファイル名の後にディレクトリ名を入れると、指定したディレクトリ内に同名でコピーができます
:::

### 削除系
- 1つのファイルを削除する（あまりやらないかも）
```
rm （ファイル名）
```
**r**e**m**oveの略

- ディレクトリごと削除する（多分どちらかというとこっちを使う）
```
rm -rf （ディレクトリ名）
```
「-rf」はオプション。この場合はディレクトリとその中身を警告なしで消す、という意味


:::message alert
#### 注意点
- 削除してしまうと元に戻せないので、再生成されるディレクトリに限定して使う方がいい
- ファイル1つだけを消すなら、コマンドではなく右クリックで消すほうが安全かも
:::
